# ComputorV1

####Usage
```
Usage: ./ComputorV1 "expression" [-d]
-h, --help			display this help and exit
-d, --debug			debug mode
```

#### Программа решения квадратных уравнений

Порядок выполнения решения уравнения.
1. Токенизация на простые токены `RealNumber, Operator, Variable, Space`.
2. Простая проверка на наличие знака `=` и количество переменных. По-умолчанию должна быть одна переменная.
3. Разделение уравнения на две части: левую и правую от знака равно. Далее для каждой части осуществляются следующие действия:
    - перевод токенов в обратную польскую нотацию `RPN`.
    - из токенов в `RPN` создается дерево вычисления типа `Expression`. 
    - производится упрощение `Expression`, т.е. производятся <b>возможные</b> операции, например складываются числа, 
    переменная умножается или делится на число и т.д. На данном этапе производится не полное упрощение, а только рядом 
    стоящих членов.
4. После упрощения в качестве бинарных операторов должны остаться только `+` и `-`. 
5. Далее правую часть переносим в левую со сменой знаков. В правой остается только ноль.
6. Операторы вычитания меняем на операторы сложения, при этом меняя знак правого члена.
7. Сортируем и приводим в вид, где старший член многочлена располагается вначале уравнения.
8. Еще раз производим упрощение, если это возможно.
9. Далее производим проверку на возможность решения.
10. <a href="https://ru.wikipedia.org/wiki/Квадратное_уравнение">Решаем квадратное уравнение</a>.
